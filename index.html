Projects

Java Birthday Paradox Tester

Motivation of the project I wanted to create a project that would test the birthday paradox. This paradox states that the probabiliy of two people sharing a birthday is over half if there are at least 23 people. This paradox appears counterintuative because people often assume that because there are 365 possible birthdays many more than 23 people are required to achieve that probability.

Functional requirements This project runs a series of trials to test this paradox, creating random birthdays and testing similarity among them. In each trial, the number of participants increases, resulting in a different probability of achieving the same birthday.

Project design This project was broken down into a Birthday class, a Trial class, and an Experiment class, and run through a Simulator class in order to efficiently conduct many experiements with multiple trials.

Implementation I engaged with test driven development to implement to Birthday and Simulator classes, as well as their testing classes. This was my first experience with test driven development and I learned how to efficiently write testable code.

Elevator Simulator

Motivation of the project I wanted to create a project that would simulate different behaviors of elevators. This project aims to demostrate the differing behaviors of different types of elevators and their abilities to move between floors executing calls

Functional requirements This project reads in elevator call requests from a file and uses a scheduler to assign different elevators to different requests

Project design This project was broken down into 4 elevator classes, representing the 4 kinds of elevators, that extend an abstract class, as well as Call, Scheduler, and Simulator classes to simulate the other required parts of the project

Implementation I engaged with test driven development to implement this project. This was done to ensure clean and efficient code that met the desired guidlines.

NBA Simulator

Motivation of the project We wanted to create a simulation of what it is like to run an NBA franchise, complete with different times in the season and different actions available to the user

Functional requirements This project creates a simulation that puts the user in charge of a team and allows them to perform actions like signing and drafting players.

Project design This project was broken down into multiple classes representing teams, players, a league, and several other classes that contain the actions available.

Implementation I engaged with test driven development with my team members to efficiently develop code and add new features. A key aspect of our design was the ability to add features in the future. We hope to next implement a minigame style option to interact more with the team during the season
